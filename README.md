# Notes

This repository uses a Markdown-first note system. Notes stay as plain files and folders, and uncertain items should go into `00_inbox/` before being sorted elsewhere.

## Structure

- `00_inbox/`: quick capture, unsorted notes, and migration/admin notes
- `01_projects/`: project-specific working notes and research folders
- `02_reference/`: reusable technical reference material
- `03_life/`: personal notes
- `90_archive/`: inactive or historical material
- `assets/`: shared attachments that are not tied to a single project folder

## Conventions

- Prefer conservative `snake_case` names for new files and folders.
- Keep note prose intact; rename mainly for clarity or obvious spelling fixes.
- Keep project-local images and attachments inside the project folder when they belong there.
- Add a minimal `README.md` inside larger project folders when context is useful.
